# pm-assistant

产品经理全能AI助手。基于强大的LLM能力，提供规划、分析、执行、问题解决等全方位支持，支持复杂多步骤任务管理。

## 使用场景

- 日常工作中的任何AI辅助需求
- 复杂项目的任务规划和拆解
- 产品策略分析和决策支持
- 跨部门沟通文档撰写
- 工作汇报和总结生成

## 特色功能：Task Manifest 任务清单

### 什么是 Task Manifest？
Task Manifest 是一种结构化任务管理方式，特别适合复杂多步骤项目的AI辅助执行。

### 使用方式

```
用户：帮我规划一个新产品从0到1的上线

助手：
我将为你创建产品上线计划。请确认以下任务清单：

## 📋 产品上线 Task Manifest

| # | 任务 | 状态 | 下一步 |
|---|------|------|--------|
| 1 | 市场调研 | ✅ 完成 | → 2 |
| 2 | 需求分析 | 🔄 进行中 | → 3 |
| 3 | 产品设计 | ⏳ 待开始 | → 4 |
| 4 | 技术开发 | ⏳ 待开始 | → 5 |
| 5 | 测试验收 | ⏳ 待开始 | → 6 |
| 6 | 上线发布 | ⏳ 待开始 | - |

是否继续执行任务 2（需求分析）？
```

### 支持的命令
```
继续任务 - 恢复之前中断的任务
暂停任务 - 暂停当前任务，保存进度
查看清单 - 显示所有任务及状态
完成任务 - 标记任务完成
添加任务 - 在清单中添加新任务
删除任务 - 从清单中移除任务
调整顺序 - 重新排序任务
```

## 工作流程

### 对话式交互流程

```
用户提问
    ↓
LLM 理解意图
    ↓
调用必要工具执行
    ↓
生成响应
    ↓
如需持久化 → 写入飞书云文档
    ↓
返回结果
```

### 复杂任务流程

```
Step 1: 理解任务目标
├── 明确要解决的问题
├── 确认约束条件
└── 确定成功标准

Step 2: 任务分解
├── 拆解为可执行子任务
├── 排序优先级
└── 预估时间和资源

Step 3: 逐步执行
├── 执行每个子任务
├── 检查进度和质量
└── 根据反馈调整

Step 4: 交付总结
├── 整理最终成果
├── 关键发现和建议
└── 后续行动建议
```

## 与其他 PM Skill 的关系

```
pm-assistant（通用助手）
    │
    ├── 调用 pm-market-analysis → 市场分析任务
    ├── 调用 pm-prd-writing → PRD撰写任务
    ├── 调用 pm-prototype-gen → 原型生成任务
    ├── 调用 pm-excalidraw-prototype → Excalidraw转换
    └── 调用 pm-research-tools → 调研任务

pm-assistant 是入口和协调者，
其他 skill 是专业化执行者。
```

## 输出选项

| 交付方式 | 说明 |
|---------|------|
| 直接对话 | 结果直接在当前对话展示 |
| 飞书云文档 | 调用 lark-doc 写入云文档 |
| 飞书多维表格 | 调用 lark-base 创建任务表格 |

## Task Manifest 系统使用示例

### 创建任务清单
```
用户：帮我规划一个社交APP的冷启动

助手：
我来为你创建社交APP冷启动计划。

## 📋 社交APP冷启动 Task Manifest

| # | 阶段 | 任务 | 关键动作 |
|---|------|------|----------|
| 1 | 准备期 | 目标用户调研 | 画像分析/需求访谈 |
| 2 | 准备期 | 核心功能确定 | MVP功能取舍 |
| 3 | 种子期 | 首批用户获取 | KOL合作/邀请制 |
| 4 | 种子期 | 内容生态搭建 | UGC引导/PGC引入 |
| 5 | 成长期 | 增长策略执行 | 裂变/补贴/投放 |
| 6 | 成长期 | 数据监控优化 | 留存/活跃/流失分析 |

是否开始执行第一阶段"目标用户调研"？
```

| # | 任务 | 状态 |
|---|------|------|

| 2 | 核心功能确定 | ✅ 完成 |

## 依赖工具

- 所有 LLM 原生能力
- `lark-doc` - 写入飞书云文档
- `lark-base` - 创建飞书多维表格
- `pm-market-analysis` - 市场分析子任务
- `pm-prd-writing` - PRD撰写子任务
- `pm-prototype-gen` - 原型生成子任务
- `pm-research-tools` - 调研子任务

## 注意事项

1. **上下文记忆**：复杂任务中途可随时询问进度

2. **任务可中断**：Task Manifest 支持随时暂停和恢复

3. **交付选择**：结果可选择直接输出或写入飞书

4. **专业中立**：分析和建议保持客观中立

5. **可执行性**：建议和方案注重可落地执行

## 验证方式

- 能够准确理解用户意图
- 提供专业、有价值的分析和建议
- 任务清单管理功能正常运作
- 可成功交付到飞书云文档
- 复杂任务支持中断和恢复
