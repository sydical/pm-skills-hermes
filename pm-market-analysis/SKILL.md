# pm-market-analysis

产品经理市场分析技能。输入产品/市场/竞品关键词，进行深度调研，生成结构化市场分析报告。

## 使用场景

- 新产品立项前的市场调研
- 竞品分析与发展趋势研究
- 行业环境评估（PESTEL）
- 市场规模与细分市场分析

## 输入

用户提供：
- **分析对象**：产品名、行业、市场领域
- **分析维度**（可选）：SWOT / PESTEL / Porter五力 / 全面分析
- **交付格式**：飞书云文档 / 本地Markdown

## 分析框架

### SWOT 分析
```
S - Strengths（优势）
W - Weaknesses（劣势）
O - Opportunities（机会）
T - Threats（威胁）
```

### PESTEL 分析
```
P - Political（政治）
E - Economic（经济）
S - Social（社会）
T - Technological（技术）
E - Environmental（环境）
L - Legal（法律）
```

### Porter's Five Forces
```
1. 行业竞争者威胁
2. 新进入者威胁
3. 替代品威胁
4. 供应商议价能力
5. 买家议价能力
```

## 工作流程

### Step 1: 信息收集
```
使用 browser 工具进行多源搜索：
- Bing/Google 搜索行业报告
- 抓取知乎/小红书/微博 相关讨论
- 访问行业媒体网站获取数据
```

### Step 2: 内容整理
```
将收集的信息分类整理：
- 市场规模数据
- 竞争格局信息
- 用户需求洞察
- 发展趋势信号
- 政策环境因素
```

### Step 3: 结构化分析
```
根据选定框架进行深度分析：
- 市场现状与规模
- 竞争格局地图
- 用户画像与痛点
- 技术演进趋势
- 政策影响评估
```

### Step 4: 报告生成
```
生成 Markdown 格式市场分析报告，包含：
# [产品/市场名称] 市场分析报告

## 一、执行摘要
## 二、市场概况
## 三、竞争格局分析
## 四、用户洞察
## 五、发展趋势
## 六、机会与风险
## 七、建议与结论
```

### Step 5: 交付飞书
```
调用 lark-doc skill：
1. 创建飞书云文档
2. 将报告内容写入文档
3. 返回文档链接
```

## 依赖工具

- `browser_navigate` - 访问网页
- `browser_snapshot` - 获取页面内容
- `search_files` - 本地文件搜索（已有报告）
- `lark-doc` - 写入飞书云文档

## 注意事项

1. **数据时效性**：优先获取近1年内的数据和报告
2. **信息交叉验证**：重要数据需多个来源交叉确认
3. **客观中立**：分析需呈现正反两面，避免偏见
4. **可执行性**：建议需具体可落地，避免空洞结论

## 验证方式

- 报告结构完整，包含7大核心章节
- 有具体数据支撑（市场规模、增长率等）
- 建议有优先级排序
- 成功写入飞书云文档并返回链接
